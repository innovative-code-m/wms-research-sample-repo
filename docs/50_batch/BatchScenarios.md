# バッチ処理シナリオ

## 概要

第2段階で実装した代表バッチ処理のシナリオを整理します。対象は、日次在庫集計と棚卸差異レポート作成です。

---

## 1. 日次在庫集計の処理流れ

### 入力

- 実行日
- 現在の在庫一覧
- 商品マスタ

### 処理流れ

1. バッチ実行日を受け取る
2. 全在庫を商品コード、倉庫コード、ロケーションコードの順で走査する
3. 各在庫を `DailyStockSnapshot` に変換する
4. 実行日単位でスナップショットを保存する
5. 出力用に商品名付きの一覧を生成する
6. 件数と実行ログを返す

### 出力

| 項目 | 内容 |
|------|------|
| 集計結果 | `DailyStockSnapshotView` の一覧 |
| 件数 | スナップショット件数 |
| ログ | 実行日と件数を含むログ文字列 |

### 異常系

| ケース | 挙動 |
|------|------|
| 在庫データが空 | 0 件の結果を返す |
| 商品名が解決できない | 商品コードは出力し、商品名は `(unknown item)` とする |

### 研究上の意味

- 在庫の現時点断面を固定化するため、将来のファイル出力型バッチへの変換対象に向く
- オンライン更新後の状態を、バッチ側の参照専用処理として切り出している

---

## 2. 棚卸差異レポート作成の処理流れ

### 入力

- 棚卸日
- 棚卸実績一覧
- 理論在庫一覧
- 商品マスタ

### 処理流れ

1. 指定日の棚卸実績を取得する
2. 各棚卸実績に対し、同一キーの理論在庫を検索する
3. 差異 `実棚数 - 理論在庫` を算出する
4. `InventoryDifference` を生成する
5. 商品名付きの差異一覧を作成する
6. 全件数と差異あり件数、実行ログを返す

### 出力

| 項目 | 内容 |
|------|------|
| 差異レポート | `InventoryDifferenceView` の一覧 |
| 差異件数 | 差異が 0 でない行数 |
| 実行ログ | 棚卸日、行数、差異件数を含むログ文字列 |

### 異常系

| ケース | 挙動 |
|------|------|
| 棚卸実績が空 | 0 件のレポートを返す |
| 対応する理論在庫が存在しない | 理論在庫を 0 とみなして差異を算出する |
| 商品名が解決できない | 商品コードは出力し、商品名は `(unknown item)` とする |

### 研究上の意味

- 棚卸実績と理論在庫の比較は、保証空間研究における整合性確認の典型例になる
- 将来のCOBOLバッチ模倣では、入力ファイル読込、突合、差異出力の手続型分割に落とし込みやすい

---

## 入力 / 出力 / ログの対応

| バッチ | 入力 | 出力 | ログ |
|--------|------|------|------|
| 日次在庫集計 | `Stock`, `Item`, 実行日 | 日次スナップショット一覧 | 実行日、件数 |
| 棚卸差異レポート | `InventoryCount`, `Stock`, `Item`, 棚卸日 | 差異一覧 | 棚卸日、全件数、差異件数 |
