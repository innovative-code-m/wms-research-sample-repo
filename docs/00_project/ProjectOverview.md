# プロジェクト概要

## このシステムの目的

本システムは、**COBOL資産の移行設計・構造解析・保証空間研究**のための、中規模WMS（Warehouse Management System）サンプルです。

本番業務システムではなく、以下の目的を持つ研究用の具体例モデルです。

- 移行対象の実例を、公開可能な形で持つ
- 画面、帳票、バッチ、マスタ、トランザクション、在庫更新などの典型要素を含める
- COBOL的な構造を後から投入・模倣できるようにする
- AST/IR/CFG/DFG、責務境界、保証空間、移行難易度評価の実験対象とする

---

## 想定利用者

| 利用者 | 想定用途 |
|--------|----------|
| 移行設計研究者 | 現行業務構造の分析、移行難易度評価、再設計候補の抽出 |
| 構造解析研究者 | モジュール依存、責務境界、保証単位の分析 |
| 保証空間研究者 | 在庫更新などのトランザクション保証の検証 |
| 教育・研修 | レガシー移行の具体例としての教材 |

---

## 対象業務

- **マスタ管理**: 商品、倉庫、ロケーション、取引先
- **在庫管理**: 在庫照会、入荷、出荷、引当、棚卸
- **トランザクション**: 入荷登録、出荷指示、引当処理、出荷確定
- **帳票・バッチ**: 各種一覧、日次集計、差異レポート

---

## 想定しない業務

- 実在企業の特定業務フロー
- 業界固有の複雑なルール（医薬品GxP、食品トレーサビリティ等の詳細）
- 本番運用レベルの性能・可用性要件
- 外部システムとのリアルタイム連携

## 中規模とみなす理由

- **機能数**: オンライン10前後、バッチ4前後、帳票5前後 → 単一モジュールでは収まらない
- **責務分割**: マスタ管理、在庫更新、引当、棚卸など責務が明確に分かれる
- **データモデル**: エンティティ13以上 → 関連が複雑になり、依存解析の対象となる
- **COBOL模倣**: 将来的にプログラム単位（画面・帳票・バッチ）に分割しやすい規模感

## 将来の研究接続

| 研究テーマ | 接続ポイント |
|------------|--------------|
| COBOL構造模倣 | オンライン/バッチ/帳票をCOBOLプログラム単位にマッピング |
| AST/IR | 処理フローを構文木・中間表現に対応づけ |
| 保証空間 | 在庫更新、引当、棚卸のトランザクション境界を分析 |
| 移行難易度 | 依存関係・責務境界から難易度スコアを算出 |
| 再設計候補 | 密結合箇所、責務混在箇所の抽出 |
