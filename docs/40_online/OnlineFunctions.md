# オンライン処理一覧

## 概要

本WMSサンプルに含まれるオンライン処理の仕様です。各処理の目的、入力、処理概要、更新対象、エラー例、画面責務を定義します。

---

## 1. 商品マスタ登録

| 項目 | 内容 |
|------|------|
| 目的 | 商品の新規登録・更新・削除（論理削除） |
| 入力項目 | 商品コード、商品名、規格、単位、ステータス |
| 処理概要 | 入力内容を検証し、Item に登録または更新 |
| 更新対象 | Item |
| エラー例 | 商品コード重複、必須項目未入力 |
| 画面責務 | 入力フォーム、検証メッセージ表示、登録完了表示 |

---

## 2. 在庫照会

| 項目 | 内容 |
|------|------|
| 目的 | 指定条件での在庫一覧を表示 |
| 入力項目 | 倉庫、商品、ロケーション、ロット（いずれも任意） |
| 処理概要 | Stock を条件で検索し、一覧表示。更新なし |
| 更新対象 | なし |
| エラー例 | 該当データなし |
| 画面責務 | 検索条件入力、一覧表示、ソート・フィルタ |

---

## 3. 入荷登録

| 項目 | 内容 |
|------|------|
| 目的 | 入荷実績を登録し、在庫を増加させる |
| 入力項目 | 入荷予定ID、実数量、ロット番号、倉庫、ロケーション |
| 処理概要 | InboundResult を作成し、該当 Stock の Quantity を増加 |
| 更新対象 | InboundResult, Stock |
| エラー例 | 入荷予定なし、数量0以下、在庫レコード未存在 |
| 画面責務 | 入荷予定選択、実績入力、登録実行、結果表示 |

---

## 4. 出荷指示登録

| 項目 | 内容 |
|------|------|
| 目的 | 出荷指示を新規登録する |
| 入力項目 | 得意先、商品、数量、希望出荷日 |
| 処理概要 | OutboundOrder を新規作成。ステータスは「未引当」 |
| 更新対象 | OutboundOrder |
| エラー例 | 得意先・商品マスタなし、数量0以下 |
| 画面責務 | 入力フォーム、登録実行、出荷指示番号表示 |

---

## 5. 引当確認

| 項目 | 内容 |
|------|------|
| 目的 | 出荷指示に対する引当状況を照会 |
| 入力項目 | 出荷指示番号 |
| 処理概要 | Allocation を検索し、引当一覧を表示。更新なし |
| 更新対象 | なし |
| エラー例 | 出荷指示なし |
| 画面責務 | 出荷指示番号入力、引当一覧表示 |

---

## 6. 引当処理

| 項目 | 内容 |
|------|------|
| 目的 | 出荷指示に対する在庫引当を実行 |
| 入力項目 | 出荷指示番号（自動引当の場合はシステムが在庫を選択） |
| 処理概要 | 在庫から引当可能数量を確保し、Allocation を作成。Stock の ReservedQuantity を更新 |
| 更新対象 | Allocation, Stock |
| エラー例 | 在庫不足、出荷指示が既に引当済み |
| 画面責務 | 引当実行、結果表示、エラー時の在庫不足メッセージ |

---

## 7. 出荷確定

| 項目 | 内容 |
|------|------|
| 目的 | 出荷実績を確定し、在庫を減らす |
| 入力項目 | 出荷指示番号、実出荷数量 |
| 処理概要 | Shipment を作成し、Stock の Quantity と ReservedQuantity を減算 |
| 更新対象 | Shipment, Stock, OutboundOrder（ステータス更新） |
| エラー例 | 引当未済、実出荷数が引当数を超過 |
| 画面責務 | 出荷指示選択、実出荷数入力、確定実行、結果表示 |

---

## 8. 棚卸入力

| 項目 | 内容 |
|------|------|
| 目的 | 棚卸で計上した実数量を登録 |
| 入力項目 | 倉庫、ロケーション、商品、実数量、棚卸日 |
| 処理概要 | InventoryCount を作成。在庫は更新しない（差異は別処理で確認） |
| 更新対象 | InventoryCount |
| エラー例 | 倉庫・ロケーション・商品の組み合わせ不正、数量0未満 |
| 画面責務 | 棚卸データ入力、一括登録、登録結果表示 |

---

## 9. 差異確認

| 項目 | 内容 |
|------|------|
| 目的 | 棚卸実績と帳簿在庫の差異を照会 |
| 入力項目 | 棚卸ID、倉庫、ロケーション（任意） |
| 処理概要 | InventoryCount と Stock を突合し、差異一覧を表示 |
| 更新対象 | なし |
| エラー例 | 棚卸データなし |
| 画面責務 | 検索条件入力、差異一覧表示、差異数量のハイライト |
