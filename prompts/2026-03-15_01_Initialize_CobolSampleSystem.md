# 目的

新規リポジトリ `wms-research-sample-repo` を初期化してください。

このリポジトリは、COBOL資産の移行設計・構造解析・保証空間研究のために使用する、
**中規模のWMS（Warehouse Management System）サンプルシステム**の公開用研究対象リポジトリです。

これは本番業務システムではなく、以下の目的を持つ **研究用の具体例モデル** です。

- 移行対象の実例を、公開可能な形で持つ
- 画面、帳票、バッチ、マスタ、トランザクション、在庫更新などの典型要素を含める
- COBOL的な構造を後から投入・模倣できるようにする
- AST/IR/CFG/DFG、責務境界、保証空間、移行難易度評価の実験対象とする
- 将来的に「現行業務構造」「移行設計」「再設計」の比較対象とする

この初期化では、**実装を大量に書くことが目的ではない**。  
まずは研究対象として扱いやすいように、リポジトリの骨格、文書、構成、最小限のサンプルコードを整備してください。

---

# このプロジェクトで重視すること

1. **研究対象としての扱いやすさ**
   - 業務責務が明確
   - 画面/帳票/バッチ/マスタ/在庫更新などの構造が分かる
   - 将来、構造解析や依存解析をしやすい

2. **公開可能なサンプルであること**
   - 実在企業依存の内容は禁止
   - 汎用的なWMSの業務モデルにする
   - データ項目・帳票名・処理名は一般化する

3. **中規模システムらしさ**
   - 小さすぎるサンプルではなく、業務責務の分割が必要な規模感
   - ただし初期段階では「完成品」にしない
   - 将来段階的に拡張できる構造を優先

4. **後のCOBOL化・移行研究への接続性**
   - オンライン処理
   - バッチ処理
   - 帳票
   - マスタ参照
   - 在庫更新
   - 入出庫・引当・棚卸などの業務イベント  
   これらを含められる構造にする

# システムの位置づけ

このリポジトリは「業務用サンプル」であり、以下の3層で研究に使えるようにしてください。

1. **業務構造を見る層**
   - どの機能がどの責務を持つか
   - どのマスタ・トランザクションを扱うか

2. **プログラム構造を見る層**
   - モジュール分割
   - 依存方向
   - バッチとオンラインの分離
   - 入出力境界

3. **移行研究に使う層**
   - COBOL的責務に分割しやすい
   - 再設計対象として論点化しやすい
   - 将来、AST/IR/保証単位を対応づけやすい
